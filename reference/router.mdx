Rehike's routing system is split across two files: `modules/Rehike/ControllerV2/Router.php` contains the `Router` class implementation, and `router.php` in the project root is the configuration table that calls it. When the `Bootloader` finishes its setup tasks, it `require`s `router.php`, which in turn calls the four static methods described on this page — `funnel()`, `redirect()`, `get()`, and `post()` — to register all routes for the current request.

## Pattern syntax

Before diving into each method, here is the full set of wildcard tokens the router understands. All patterns are matched against the request URI path (the query string is stripped before comparison).

| Token | Meaning |
|---|---|
| `*` | Matches exactly one path segment (any characters except `/`). |
| `**` | Matches one or more path segments (crosses `/` boundaries). |
| `(*)` | Capture group — matches one path segment and makes it available as `$1`, `$2`, etc. in redirect target strings. |
| `~/path` | Exclusion prefix used inside `funnel()`. A pattern starting with `~` means "do **not** funnel this path", even if a broader pattern would otherwise match it. |
| `default` | Special key used as the map key in `get()` and `post()` arrays. Acts as the catch-all when no other pattern matches. |

## Router::funnel()

```php
public static function funnel(array $paths): void
```

Passes matching requests directly through to `www.youtube.com` without any Rehike processing. The request URI is replayed verbatim against the upstream YouTube server and the response is streamed back to the client. This is used for API endpoints, static resources, and account management pages that Rehike does not need to intercept.

<ParamField path="$paths" type="string[]" required>
  An array of URL glob patterns to proxy. Any pattern prefixed with `~` is treated as an **exclusion**: if the current URI matches an exclusion, the funnel returns immediately and Rehike processes the request normally. Exclusions must appear before the broader pattern they narrow.
</ParamField>

### How it works internally

`Router::funnel()` iterates through `$paths` in order. For each entry it checks:

1. If the pattern starts with `~`, test the substring after `~` against the current URI. If it matches, **return early** — the request falls through to Rehike.
2. Otherwise, if the pattern matches the current URI, call `SimpleFunnel::funnelCurrentPage()` to proxy the request.

### Example — from `router.php`

```php
Router::funnel([
    "/api/*",
    "~/youtubei/v1/player",         // exclude player — proxied specially below
    "/youtubei/v1/player/heartbeat",
    "/youtubei/*",                  // all /youtubei/ except the excluded player
    "/s/*",
    "/embed/*",
    "/yts/*",
    "/favicon.ico",
    "/subscribe_embed",
    "/login",
    "/logout",
    "/signin",
    "/upload",
    "/t/*",
    "/account",
    "/account_notifications",
    "/account_playback",
    "/account_privacy",
    "/post/*",
    "/feeds/*",
    "/img/*",
    "/yt*",
    "/about/",
    // ... additional entries
]);
```

The exclusion `~/youtubei/v1/player` ensures that even though `/youtubei/*` would match that path, Rehike gets to handle it via a dedicated proxy controller registered in `Router::post()`.

## Router::redirect()

```php
public static function redirect(array $rules): void
```

Declares URL rewrite rules. When a request URI matches a rule's pattern, the router issues an HTTP redirect (or invokes a callable to compute the target) before any controller runs. Rules are evaluated in order and the first match wins.

<ParamField path="$rules" type="array" required>
  An associative array mapping URL glob patterns to redirect targets. Each value is either a **string** (with optional `$1`/`$2` capture-group references) or a **callable** that receives a `RequestMetadata` object and returns the target URL string.
</ParamField>

### String targets

For string targets the matched capture groups replace `$1`, `$2`, etc. in the target:

```php
"/watch/(*)" => "/watch?v=$1"
// /watch/dQw4w9WgXcQ  →  /watch?v=dQw4w9WgXcQ
```

### Callable targets

For more complex redirects, pass an anonymous function. It receives a `RequestMetadata` instance and must return a string URL:

```php
"/shorts/(*)" => function($request) {
    if (isset($request->path[1]))
        return "/watch?v=" . $request->path[1];
    else
        return "/watch";
},
```

### Example — from `router.php`

```php
Router::redirect([
    "/watch/(*)"             => "/watch?v=$1",
    "/shorts/(*)"            => function($request) {
        if (isset($request->path[1]))
            return "/watch?v=" . $request->path[1];
        else
            return "/watch";
    },
    "/live/(*)"              => "/watch?v=$1",
    "/hashtag/(*)"           => "/results?search_query=$1",
    "/feed/what_to_watch/**" => "/",
    "/source/(*)"            => function($request) {
        if (isset($request->path[1]))
            return "/attribution?v=" . $request->path[1];
        else
            return "/attribution";
    },
    "/redirect(/|?)*"        => function($request) {
        if (isset($request->params->q))
            return urldecode($request->params->q);
    },
    "/feed/library"          => "/profile",
    "/feed/you"              => "/profile",
    "/subscription_manager"  => "/feed/channels",
    "/rehike/settings"       => "/rehike/config",
    "/subscription_center?(*)" => function($request) {
        if ($user = @$request->params->add_user)
            return "/user/$user?sub_confirmation=1";
        else if ($user = @$request->params->add_user_id)
            return "/channel/$user?sub_confirmation=1";
    }
]);
```

## Router::get()

```php
public static function get(array $routes): void
```

Maps GET request URL patterns to controller class names. Only executed when `$_SERVER['REQUEST_METHOD']` is `GET`. The router iterates the entire array before dispatching to ensure the **longest / most specific match** wins (not simply the first match).

<ParamField path="$routes" type="array" required>
  An associative array of `"/pattern" => ControllerClass::class` entries, plus an optional `"default"` key whose value is the controller dispatched when no pattern matches.
</ParamField>

Controllers must implement `IController` and either `IGetController` (synchronous `get()` method) or `IGetControllerAsync` (async `getAsync()` method returning a `Promise`). Passing a class that does not implement the expected interface throws an exception.

### Example — from `router.php`

```php
Router::get([
    "/"                          => FeedPageController::class,
    "/feed/**"                   => FeedPageController::class,
    "/watch"                     => WatchPageController::class,
    "/user/**"                   => ChannelPageController::class,
    "/channel/**"                => ChannelPageController::class,
    "/c/**"                      => ChannelPageController::class,
    "/live_chat"                 => special\GetLiveChatController::class,
    "/live_chat_replay"          => special\GetLiveChatController::class,
    "/feed_ajax"                 => ajax\FeedAjaxController::class,
    "/results"                   => ResultsPageController::class,
    "/playlist"                  => PlaylistPageController::class,
    "/oops"                      => OopsPageController::class,
    "/related_ajax"              => ajax\RelatedAjaxController::class,
    "/browse_ajax"               => ajax\BrowseFragmentsController::class,
    "/addto_ajax"                => ajax\AddtoFragmentsController::class,
    "/rehike/version"            => rehike\RehikeVersionPageController::class,
    "/rehike/static/**"          => rehike\StaticRouter::class,
    "/share_ajax"                => ajax\ShareFragmentsAjax::class,
    "/picker_ajax"               => ajax\PickerFragmentsAjax::class,
    "/attribution"               => AttributionPageController::class,
    "/profile"                   => ProfileRedirectEndpointController::class,
    "/channel_switcher"          => ChannelSwitcherPageController::class,
    "/rehike/config"             => rehike\RehikeConfigPageController::class,
    "/rehike/config/**"          => rehike\RehikeConfigPageController::class,
    "/rehike/ajax/git_ajax"      => rehike\ajax\RehikeVersionGitAjaxController::class,
    "/rehike/server_info"        => rehike\RehikeServerInfoPageController::class,
    "/rehike/extensions"         => rehike\RehikeExtensionsPageController::class,
    "/html5"                     => Html5PageController::class,
    "/annotations_invideo"       => ajax\AnnotationsInvideoController::class,
    "/get_video_metadata"        => ajax\GetVideoMetadataController::class,
    "/pagead"                    => function() { exit(); },
    "/player_204"                => function() { exit(); },
    "/generate_204"              => function() { exit(); },
    "default"                    => ChannelPageController::class,
]);
```

<Tip>
  The `"default"` route is intentionally set to `ChannelPageController`. YouTube supports short-form username URLs like `/PewDiePie` that resolve to `/user/PewDiePie`. Any URL that doesn't match a known route is handed to the channel controller, which performs the lookup and shows the 404 page if the channel doesn't exist.
</Tip>

## Router::post()

```php
public static function post(array $routes): void
```

Maps POST request URL patterns to controller class names. Identical in structure to `Router::get()` but only activated when `$_SERVER['REQUEST_METHOD']` is `POST`.

<ParamField path="$routes" type="array" required>
  An associative array of `"/pattern" => ControllerClass::class` entries, plus an optional `"default"` key. Controllers must implement `IPostController` or `IPostControllerAsync`.
</ParamField>

### Example — from `router.php`

```php
Router::post([
    "/youtubei/v1/player"          => special\InnertubePlayerProxyController::class,
    "/feed_ajax"                   => ajax\FeedAjaxController::class,

    "/subscription_ajax"           => ajax\SubscriptionAjaxController::class,
    "/service_ajax"                => ajax\ServiceAjaxController::class,
    "/comment_service_ajax"        => ajax\CommentServiceAjaxController::class,
    "/addto_ajax"                  => ajax\AddtoFragmentsController::class,
    "/live_events_reminders_ajax"  => ajax\LiveEventsRemindersAjaxController::class,
    "/delegate_account_ajax"       => ajax\DelegateAccountFragmentsController::class,
```

## The RequestMetadata object

Every callable controller value — and every redirect callable — receives an instance of `Rehike\ControllerV2\RequestMetadata`. It exposes the following properties:

| Property | Type | Description |
|---|---|---|
| `$request->method` | `string` | The HTTP method (`"GET"` or `"POST"`). |
| `$request->path` | `string[]` | The request path split on `/`. Index `0` is the first segment, `1` the second, and so on. |
| `$request->rawPath` | `string` | The full raw request URI including query string. |
| `$request->headers` | `EasyAccessMap` | All incoming HTTP headers, accessible as `$request->headers["Header-Name"]`. |
| `$request->params` | `EasyAccessMap` | GET query parameters, accessible as `$request->params->paramName` or `$request->params["paramName"]`. |
| `$request->body` | `mixed` | POST body, decoded from JSON or URL-encoded form data. Only present on POST requests. |

### Usage in a redirect callable

```php
"/subscription_center?(*)" => function($request) {
    if ($user = @$request->params->add_user)
        return "/user/$user?sub_confirmation=1";
    else if ($user = @$request->params->add_user_id)
        return "/channel/$user?sub_confirmation=1";
}
```

### Usage in a path-based redirect

```php
"/shorts/(*)" => function($request) {
    // $request->path[1] is the video ID segment after /shorts/
    if (isset($request->path[1]))
        return "/watch?v=" . $request->path[1];
    else
        return "/watch";
},
```

## Controller class references

Controller values in `get()` and `post()` arrays are passed as fully-qualified PHP class names using the `::class` constant syntax. The router resolves the class at dispatch time, verifies that it implements the correct controller interface, instantiates it, calls `initializeController(new RequestMetadata())`, and then invokes `get()` / `getAsync()` / `post()` / `postAsync()` as appropriate.

```php
// Correct — uses ::class to produce a fully-qualified class name string
"/results" => ResultsPageController::class,

// Also valid for quick exit routes — plain callables are supported
"/pagead"  => function() { exit(); },
```

All classes in the `Rehike\Controller\` namespace are resolved by the autoloader from the `controllers/` directory. Sub-namespaces like `ajax\` and `special\` map to sub-directories within `controllers/`.
