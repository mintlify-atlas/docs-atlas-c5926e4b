InnerTube is YouTube's private internal API — the same API that YouTube's own web, mobile, and TV frontends use to fetch page data. It is not publicly documented, but its request and response format is consistent enough for Rehike to rely on it as its primary data source. Every piece of live content that Rehike renders — video metadata, search results, channel feeds, comments, and more — arrives as an InnerTube JSON response.

Rehike communicates with InnerTube through the `Network` class (`modules/Rehike/Network.php`), which wraps the low-level `NetworkCore` HTTP layer and handles context injection, authentication forwarding, DNS configuration, and request logging.

## InnerTube endpoint

All InnerTube actions are reached by POSTing to:

```
https://www.youtube.com/youtubei/v1/{action}?key={apiKey}
```

The action name selects which type of data to retrieve. Rehike uses the following actions across its controllers:

| Action | Used by | Returns |
|---|---|---|
| `browse` | `FeedPageController`, `ChannelPageController` | Home feed, channel pages, playlist browse |
| `next` | `WatchPageController` | Watch page data (metadata, comments, related videos) |
| `player` | `WatchPageController` | Streaming URLs, player config, captions |
| `search` | `ResultsPageController` | Search results |

## Client context

Every InnerTube request body must include a `context` object that identifies the client making the request. Rehike uses the `WEB` client, mirroring what YouTube's own website would send. The context is generated by `InnertubeContext::generate()`:

```php
// modules/Rehike/InnertubeContext.php
public static function generate(
    string|int $cname,
    string $cver,
    ?string $visitorData = null,
    ?string $hl = null,
    ?string $gl = null
): object {
    return (object) [
        'context' => (object) [
            'client' => (object) [
                'hl'            => $hl,   // host language, e.g. "en"
                'gl'            => $gl,   // geolocation, e.g. "US"
                'visitorData'   => self::genVisitorData($visitorData),
                'clientName'    => $cname,   // "WEB"
                'clientVersion' => $cver,    // "2.20250928.01.00"
                'userAgent'     => $_SERVER['HTTP_USER_AGENT'] ?? ""
            ],
            'user' => (object) [
                'lockedSafetyMode' => false
            ],
            'request' => (object) [
                'useSsl'                  => true,
                'internalExperimentFlags' => [],
                'consistencyTokenJars'    => []
            ]
        ]
    ];
}
```

The `visitorData` field is a Protocol Buffers-encoded string containing the `VISITOR_INFO1_LIVE` cookie value and the current Unix timestamp. Rehike encodes this in `InnertubeContext::genVisitorData()` and forwards it as both a context field and as the `X-Goog-Visitor-Id` request header.

## `Network::innertubeRequest()`

The primary method for all InnerTube calls. Its full signature as defined in `modules/Rehike/Network.php`:

```php
public static function innertubeRequest(
    string $action,
    array  $body            = [],
    string $clientName      = "WEB",
    string $clientVersion   = "2.20250928.01.00",
    bool   $ignoreErrors    = false,
    bool   $useAuthentication = true
): Promise /*<IResponse>*/
```

| Parameter | Type | Default | Purpose |
|---|---|---|---|
| `$action` | `string` | — | InnerTube action name, e.g. `"browse"`, `"next"`, `"player"` |
| `$body` | `array` | `[]` | Request payload merged with the generated client context |
| `$clientName` | `string` | `"WEB"` | InnerTube client identifier |
| `$clientVersion` | `string` | `"2.20250928.01.00"` | Client version string |
| `$ignoreErrors` | `bool` | `false` | If `true`, resolve the promise even on non-200 responses |
| `$useAuthentication` | `bool` | `true` | Whether to attach auth headers from the sign-in service |

The method merges the caller's `$body` with the generated context object, then fires a POST request with the following headers:

```php
$requestHeaders = [
    "Content-Type"       => "application/json",
    "X-Goog-Visitor-Id"  => InnertubeContext::genVisitorData(ContextManager::$visitorData)
];

// Authentication headers are appended when $useAuthentication === true
// and a user is signed in (see Authentication section below).
```

The method returns a `Promise<IResponse>`. Awaiting it with `yield` (inside an `async()` coroutine) gives you the response object; call `->getJson()` to decode the body:

```php
// Typical usage in a controller (WatchPageController.php)
$nextRequest = Network::innertubeRequest(
    "next",
    ["videoId" => $yt->videoId]
);

$playerRequest = Network::innertubeRequest(
    "player",
    [
        "videoId" => $yt->videoId,
        "playbackContext" => [
            "contentPlaybackContext" => (object) [
                "html5Preference"    => "HTML5_PREF_WANTS",
                "playerHeightPixels" => 1080,
                "playerWidthPixels"  => 1920,
                "signatureTimestamp" => $yt->playerConfig->signatureTimestamp
            ]
        ]
    ],
    "WEB",
    "2.20251014.00.00"
);

$responses = yield Promise::all([
    "next"   => $nextRequest,
    "player" => $playerRequest,
]);

$nextResponse   = $responses["next"]->getJson();
$playerResponse = $responses["player"]->getJson();
```

<Note>
  All InnerTube requests in Rehike are fired concurrently using `Promise::all()`. The event loop resolves them in parallel before the controller continues, minimising total wall-clock latency per page render.
</Note>

## Authentication

When a user is signed in to Rehike, their YouTube session cookies are forwarded to every InnerTube request. The auth service registers itself with the network layer during boot via `Network::useAuthService()` or `Network::useAuthService2()`, which populates the `$innertubeAuthHeaders` array:

```php
// Set by Network::useAuthService() when AuthManager::shouldAuth() is true
$innertubeAuthHeaders = [
    "Authorization" => AuthManager::getAuthHeader(),  // SAPISIDHASH
    "Origin"        => "https://www.youtube.com",
    "Host"          => "www.youtube.com",
    "Cookie"        => Network::getCurrentRequestCookie()
];
```

For brand accounts (YouTube channels separate from the Google account), an additional GAIA ID header is appended:

```php
// Set by Network::useAuthGaiaId() for brand accounts
$innertubeAuthHeaders += [
    "X-Goog-AuthUser" => "0",
    "X-Goog-PageId"   => $gaiaId
];
```

These headers are attached to every InnerTube request where `$useAuthentication === true` (the default). If authentication fails, `Network::rejectAuthService()` clears the array so that subsequent requests proceed unauthenticated.

## The `/youtubei/v1/player` special case

The player endpoint (`/youtubei/v1/player`) is excluded from the general `youtubei/*` funnel rule in `router.php` using the `~` negation prefix:

```php
Router::funnel([
    "~/youtubei/v1/player",  // ← excluded from funnelling
    "/youtubei/*",           // all other youtubei/ requests pass through
    // ...
]);

Router::post([
    "/youtubei/v1/player" => special\InnertubePlayerProxyController::class,
    // ...
]);
```

`InnertubePlayerProxyController` intercepts player requests from the in-page JavaScript player, applies any Rehike-specific processing (e.g. ad removal, annotation injection), and then forwards the modified request to YouTube's InnerTube API before returning the response.

## `Network::dataApiRequest()`

For endpoints that require the public YouTube Data API v3 rather than InnerTube, Rehike provides `dataApiRequest()`. This uses a separate API host and key, and targets `https://www.googleapis.com/youtube/v3/{action}`:

```php
public static function dataApiRequest(
    string $action,
    array  $params,
    bool   $post = false
): Promise /*<IResponse>*/
```

This method is used in contexts where InnerTube does not expose the required data, for example fetching supplementary channel or video information via the public API.

## DNS configuration

All InnerTube requests use a configurable DNS resolver to avoid relying on the system default. The DNS address is read from the `advanced.dnsAddress` configuration property and falls back to Cloudflare's `1.1.1.1` if not set:

```php
$desiredDns = Config::getConfigProp("advanced.dnsAddress") ?? "1.1.1.1";

$request = NetworkCore::request2(
    "{$host}/youtubei/v1/{$action}?key={$key}",
    [
        "headers"     => $requestHeaders,
        "method"      => "POST",
        "body"        => json_encode($body),
        "dnsOverride" => $desiredDns,
        // ...
    ]
);
```

<Tip>
  If you are running Rehike behind a corporate proxy or in a network environment where `1.1.1.1` is blocked, set `advanced.dnsAddress` in your Rehike configuration to an accessible DNS resolver.
</Tip>

## Request logging

Every call to `innertubeRequest()` is appended to an in-memory session log (`$sessionRequestLog`) that records the action, client name, client version, authentication state, and the promise itself. This log is accessible via `Network::getSessionRequestLog()` and is surfaced in the Rehike debugger panel for development use.

```php
self::$sessionRequestLog[] = [
    "type"              => "innertube",
    "action"            => $action,
    "body"              => (object)$body,
    "clientName"        => $clientName,
    "clientVersion"     => $clientVersion,
    "ignoreErrors"      => $ignoreErrors,
    "useAuthentication" => $useAuthentication,
    "INTERNAL_promise"  => $result
];
```

## Fake InnerTube requests (development)

During development, `Network::innertubeRequestFake()` can be used in place of `innertubeRequest()` to serve a local JSON file as if it were a real InnerTube response. This allows testing against saved API dumps without making live network calls. Passing `"error"` as the file path forces an `InnertubeFailedRequestException`, which is useful for testing error-handling code paths.

```php
// Replace a real request with a local JSON dump during development
$nextRequest = Network::innertubeRequestFake(
    "/path/to/dump/next_response.json",
    "next",
    ["videoId" => "dQw4w9WgXcQ"]
);
```

<Warning>
  `innertubeRequestFake()` is strictly for local development and testing. Never ship production code that calls this method, as it will serve stale data instead of live YouTube content.
</Warning>
